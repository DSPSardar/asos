Server Setup (Ubuntu VPS)
SSH into your server, then run the automated setup script. Skip this step if using Railway — go to Step 4.
SSH into server
ssh root@YOUR_SERVER_IP
Upload project files from your machine
# Run on your LOCAL machine
rsync -avz --exclude node_modules --exclude .git \
  ./ root@YOUR_SERVER_IP:/opt/asos/
Run automated setup script
# On the SERVER — replace with your actual domain
bash /opt/asos/deploy/setup.sh yourdomain.com admin@yourdomain.com
DNS first: Point your domain A records to the server IP before running the setup script. Certbot needs to verify domain ownership via HTTP.

Start All Services
Docker Compose orchestrates the full stack: PostgreSQL → Redis → DB Migration → API → Workers × 2 → Nginx.
Start the full stack
cd /opt/asos

# Run migrations first
docker compose --env-file .env.production up -d migrate
sleep 10

# v1.5 Dual Agent migration — adds AiAgentLog table + Lead.humanFollowupRequired
docker exec asos_api npx prisma migrate deploy

# Start everything
docker compose --env-file .env.production up -d

# Seed demo data
docker exec asos_api node prisma/seed.js

# Check all containers are running
docker compose ps

5
Configure WhatsApp Webhook
This step connects Meta's WhatsApp Cloud API to your live server. Without this, no messages will arrive.
  1. Go to developers.facebook.com → Your App → WhatsApp → Configuration
  2. Under Webhook, click Edit
  3. Set Callback URL: https://api.yourdomain.com/webhooks/whatsapp
  4. Set Verify Token: same value as WHATSAPP_VERIFY_TOKEN in your .env
  5. Click Verify and Save — Meta will call your server to verify
  6. Under Webhook Fields, subscribe to: messages and message_status_updates
  7. Copy your Phone Number ID and Permanent Access Token
  8. In ASOS Settings → WhatsApp → paste those values and save
Test webhook verification manually
curl "https://api.yourdomain.com/webhooks/whatsapp\
  ?hub.mode=subscribe\
  &hub.verify_token=YOUR_VERIFY_TOKEN\
  &hub.challenge=test123"
# Should return: test123
Common issue: If verification fails, check that WHATSAPP_VERIFY_TOKEN in .env.production exactly matches what you entered in the Meta Developer Console (case-sensitive).
6
Verify End-to-End Flow
Send a test WhatsApp message to confirm the full pipeline works: Message → Webhook → Queue → Claude → Reply.
Watch live logs
# API logs
docker logs -f asos_api

# Worker logs (AI processing)
docker logs -f asos_worker

# All logs together
docker compose logs -f --tail=50
✅ What to look for in logs
  • 📨 "Inbound message queued" — webhook received message
  • ▶ "Processing inbound message" — worker picked up job
  • New contact created — CRM auto-created contact
  • New lead created — CRM auto-created lead
  • Claude AI decision logged — AI processed and scored
  • WA text message sent — reply sent back to WhatsApp
  • Meta CAPI event sent — attribution fired to Meta

Pre-Launch Checklist

  • DNS A records pointing to server IP
  • SSL certificates issued (Let's Encrypt)
  • All Docker containers healthy (docker compose ps)
  • Health check passes: GET /health → {"status":"ok"}
  • ANTHROPIC_API_KEY set in .env.production
  • WHATSAPP_APP_SECRET + WHATSAPP_VERIFY_TOKEN set
  • WhatsApp webhook verified in Meta Developer Console
  • messages + message_status_updates webhook fields subscribed
  • Phone Number ID + Access Token saved in ASOS Settings
  • Test message sent → AI replied in WhatsApp
  • Lead appears in ASOS Dashboard pipeline
  • Meta CAPI Lead event visible in Events Manager
  • Seed data loaded (node prisma/seed.js)
  • Admin login works at https://app.yourdomain.com
