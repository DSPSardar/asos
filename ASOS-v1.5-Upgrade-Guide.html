01 · Architecture

Two agents, one pipeline

Specialization beats a single do-everything prompt. The Qualifier runs cheap and deterministic; the Closer runs warmer and persuasive.

◆ Agent A

Qualifier AI

Analyzes the inbound message. Outputs structured lead intelligence.

  • Detects intent, urgency, business type
  • Assigns lead score 1–10
  • Classifies HOT / WARM / COLD
  • Recommends next action
// Qualifier output { "lead_status": "HOT", "score": 9, "intent": "high", "problem_summary": "...", "next_action": "send_proposal" }
Model · claude-haiku-4-5 · temp 0 · 256 tokens
passes context
◇ Agent B

Closer AI

Receives Qualifier's analysis. Writes one persuasive WhatsApp reply.

  • Calibrates tone to lead score
  • Handles objections naturally
  • Adds urgency on score ≥ 7
  • Picks closing type: soft/hard/urgent
// Closer output { "reply_message": "...", "closing_type": "hard", "urgency_trigger": "3 spots left" }
Model · claude-3-5-sonnet · temp 0.4 · 512 tokens
02 · System flow

Message → Qualifier → CRM → Closer → Send

Sequential pipeline. CRM is updated after Qualifier so even if Closer fails, the lead intelligence is captured.

1
Inbound message
WhatsApp webhook → BullMQ → conversation.worker.js
2
Qualifier AI
Parses message + history, returns {lead_status, score, intent, problem_summary, next_action}
3
CRM update
Lead row patched: aiScore, scoreLabel, intent, problemSummary, humanFollowupRequired
4
Closer AI
Receives Qualifier output → drafts reply with closing_type + urgency_trigger
5
Send via WhatsApp
Reply pushed via whatsappService.sendText() · stored as OUTBOUND message
6
Audit log
Both raw outputs persisted to ai_agent_logs for the future Analyst AI (v2)
03 · Lead scoring logic

Three buckets. Clear rules.

Score is canonical at 1–10 in the AI layer; the DB stores it as 0–100 (aiScore = score × 10) so existing dashboards keep working.

🔥 HOT
8–10
Strong buying intent · clear need · urgency · decision authority. Triggers human_followup_required.
◐ WARM
5–7
Asking questions · interested but missing 1+ BANT element. AI continues qualifying.
◯ COLD
1–4
Casual or unclear · off-topic · unqualified. Soft nurture, no urgency.
04 · Human takeover

HOT leads flag for human follow-up

When score ≥ 8 the lead is flagged but the AI keeps replying — sales humans get an alert in the dashboard, conversation isn't interrupted.

What happens Automatic

Lead flaggedhumanFollowupRequired = true
Activity logged"🔥 HOT lead — human follow-up required" appears in timeline
AI continuesCloser still sends a reply — no awkward silence for the lead
IndexedNew DB index lets the dashboard query "HOT leads needing follow-up" instantly

Hard handoff Stops AI

Trigger words detected"refund", "legal", "complaint", "angry", custom triggers
Qualifier returns next_action="handoff_human"Closer is skipped entirely
Conversation status → HUMAN_TAKEOVERaiEnabled=false. Agent inbox handles from here.
Same fallback if Qualifier or Closer crashesHard handoff is the safe failure mode
